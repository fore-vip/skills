## 🔐 认证 (Authentication)

### Open Key 获取方式

1. 在 [fore.vip](https://fore.vip) 平台注册/登录
2. 进入 **用户中心** → **API 设置**
3. 生成或复制你的 **Open Key**

### Header 格式

```http
Authorization: Bearer <your_open_key>
```

或

```http
X-Open-Key: <your_open_key>
```

### 认证要求

| 工具 | 认证 | 说明 |
|------|------|------|
| `create_kl` | 🔐 需要 | 发布产品 |
| `query_kl` | ❌ 公开 | 查询产品 |

## 📦 技能列表

### 1. product-create (v1.0.0) - 单个产品创建

**用途**: 创建单个 AI 产品，带自动查重

**工具**:
- `query_kl` - 检查产品名称是否存在
- `create_kl` - 创建产品

**工作流程**:
```
外部来源 → 查重 → 确认 → 发布
```

**适用场景**:
- 精确发布单个产品
- 需要查重保证
- 用户明确指定的产品

**文档**: [product-create/SKILL.md](product-create/SKILL.md)

---

### 2. product (v2.0.0) - 批量推送

**用途**: 批量推送多个产品，不查重

**工具**:
- `create_kl` - 批量创建产品

**工作流程**:
```
外部来源 → 批量发布 → 汇总结果
```

**适用场景**:
- 批量导入产品
- 快速推送多个产品
- 已在外部处理过查重

**文档**: [product/SKILL.md](product/SKILL.md)

## 🚀 智能发布 (Smart Publish)

### ⚠️ 重要说明

**内容来源**: 外部来源（**不是**现有产品数据库）

**原因**: 避免循环依赖（不能先查询产品再发布同样的产品）

### 内容来源

| 来源 | 说明 | 适用场景 |
|------|------|----------|
| **AI 生成** | AI 根据标签创作内容 | 通用产品发布 |
| **搜索引擎** | Google/Bing 搜索结果 | 分享真实产品 |
| **会话上下文** | 用户对话中提到的内容 | 用户描述的产品 |
| **GitHub** | 热门开源项目 | 技术类分享 |
| **Product Hunt** | 每日热门产品 | 新产品发现 |
| **其他技能** | copywriting 等技能 | 专业内容生成 |

## 🤖 单个产品创建流程 (product-create)

```
┌─────────────────────────────────────────────────────────┐
│  1. 用户请求发布产品                                      │
│     用户："发布一个 AI 工具 标签的产品"                    │
└───────────────────┬─────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────┐
│  2. Agent 选择内容来源                                    │
│     - AI 生成（默认）                                     │
│     - 搜索引擎                                          │
│     - 会话上下文                                        │
│     - GitHub/Product Hunt                               │
└───────────────────┬─────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────┐
│  3. 获取/生成产品信息                                    │
│     {name, content, pic, tag, url}                      │
└───────────────────┬─────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────┐
│  4. 🔍 检查名称是否已存在 (query_kl with name)            │
│     - 如已存在 → 跳过，返回现有产品                       │
│     - 如不存在 → 继续                                    │
└───────────────────┬─────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────┐
│  5. Agent 显示预览                                        │
│     "📦 产品预览：名称、描述、标签...                    │
│      是否确认发布？(确认/取消)"                          │
└───────────────────┬─────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────┐
│  6. 用户确认                                              │
│     用户："确认"                                          │
└───────────────────┬─────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────┐
│  7. Agent 调用 create_kl 发布                              │
│     POST /mcp/create_kl                                 │
└───────────────────┬─────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────┐
│  8. 发布成功                                              │
│     ✅ 产品 ID: xxx                                       │
│     🔗 https://fore.vip/p?id=xxx                        │
└─────────────────────────────────────────────────────────┘
```

## 📦 批量推送流程 (product)

```
┌─────────────────────────────────────────────────────────┐
│  1. 用户请求批量发布                                      │
│     用户："批量发布 10 个 AI 工具"                          │
└───────────────────┬─────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────┐
│  2. 准备产品列表（从外部来源）                            │
│     - AI 生成多个产品                                    │
│     - GitHub trending Top 10                             │
│     - 搜索结果列表                                       │
└───────────────────┬─────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────┐
│  3. 批量调用 create_kl（不查重）                          │
│     for product in products:                            │
│       create_kl(product)                                │
└───────────────────┬─────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────┐
│  4. 返回批量结果                                          │
│     ✅ 批量推送完成 (10/10)                              │
└─────────────────────────────────────────────────────────┘
```

## 🧩 可用工具

### query_kl (v0.0.4)

**认证**: ❌ 公开

| 参数 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `tag` | string | - | 产品标签 |
| `name` | string | - | **产品名称（模糊匹配）** |
| `limit` | number | 20 | 最大结果数 |
| `skip` | number | 0 | 分页 |

**注意**: `product` (批量推送) 技能不使用此工具

---

### create_kl (v0.0.3)

**认证**: 🔐 需要 Open Key

| 参数 | 类型 | 说明 |
|------|------|------|
| `name` | string | 产品名称 (≥2 字符) |
| `content` | string | 产品描述 (≥10 字符) |
| `pic` | string[] | 图片 URL |
| `tag` | string | 产品标签 |
| `hot` | number | 热度 |
| `url` | string | 外部链接 |

## 🔄 技能选择指南

### 使用 `product-create` 当：

- ✅ 发布单个产品
- ✅ 需要自动查重
- ✅ 用户明确指定某个产品
- ✅ 精确控制发布内容

### 使用 `product` 当：

- ✅ 批量发布多个产品（5-20 个）
- ✅ 已在外部处理过查重
- ✅ 快速导入产品列表
- ✅ 不介意可能的重复
