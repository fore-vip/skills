## 处理流程

### 1. 信息收集

向用户询问缺失的必需参数：

```
需要确认以下信息：
1. [参数 1]: 用途说明
2. [参数 2]: 用途说明
3. [参数 3]: 用途说明
```

### 2. 参数验证

```javascript
// 验证规则示例
if (!param1 || param1.length < 2) {
  throw new Error('参数 1 至少 2 个字符')
}
if (param2 < 0 || param2 > 100) {
  throw new Error('参数 2 范围 0-100')
}
```

### 3. 核心逻辑

```javascript
// 调用云函数/API/执行操作
const result = await doSomething({
  param1,
  param2,
  option1
})
```

### 4. 结果处理

**成功**:
```javascript
// 成功后的操作 (跳转/提示/展示)
```

**失败**:
```javascript
// 错误处理和用户提示
```

# 📖 各部分详解

## 1️⃣ Frontmatter (元数据)

```markdown
---
name: skill-name           # 技能唯一标识 (小写，连字符)
description: 简短描述       # 50 字以内，用于搜索和展示
version: 1.0.0            # 语义化版本号
author: your-name         # 作者/团队
license: MIT              # 开源协议
---
```

**作用**: 机器可读的元数据，用于技能发现和管理

---

## 2️⃣ 技能说明

**目的**: 让 AI 快速理解技能的用途

**要点**:
- 第一句说明核心功能
- 用列表列出典型使用场景
- 包含用户故事更好

**示例**:
```markdown
此技能帮助创建 AI 飞花令线下活动。

**使用场景**:
- 用户要创建新的线下活动
- 需要设置活动时间、地点、门票等信息
- 关联已有的 AI 机器人
```

---

## 3️⃣ 参数定义

### 必需参数 vs 可选参数

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| **必需参数** | 缺少则无法执行 | AI 必须询问用户 |
| **可选参数** | 有默认值 | AI 可跳过或使用默认值 |

### 参数表格要素

| 列名 | 说明 | 示例 |
|------|------|------|
| `参数` | 参数名称 (用反引号) | `kid` |
| `类型` | string/number/boolean/array/object | `string` |
| `说明` | 参数用途 | `关联的机器人 ID` |
| `验证规则` | 格式/范围/长度要求 | `长度>=2` |
| `示例` | 典型值 | `"kl_abc123"` |

---

## 4️⃣ 处理流程

### 标准四步流程

```
1. 信息收集 → 2. 参数验证 → 3. 核心逻辑 → 4. 结果处理
```

**每部分要点**:

| 步骤 | 内容 | 示例 |
|------|------|------|
| **信息收集** | 如何询问用户 | 问题列表 |
| **参数验证** | 验证规则代码 | if 判断 |
| **核心逻辑** | 主要业务逻辑 | API 调用 |
| **结果处理** | 成功/失败处理 | 跳转/提示 |

---

## 5️⃣ 错误处理

**目的**: 让 AI 知道如何处理异常情况

**表格要素**:
- **错误信息**: 实际返回的错误文本
- **原因**: 可能的触发原因
- **处理建议**: AI 应该如何引导用户

**示例**:
```markdown
| 错误信息 | 原因 | 处理建议 |
|----------|------|----------|
| `用户未登录` | token 过期 | 引导用户重新登录 |
| `机器人不存在` | kid 错误 | 帮用户查询正确的 ID |
```

---
