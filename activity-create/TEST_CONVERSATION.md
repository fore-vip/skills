## 对话 3: 错误处理 - 时间冲突

### User
帮我创建一个活动，明天下午 3 点到 2 点

### Assistant (思考过程)
用户输入的时间有问题：结束时间 (14:00) 早于开始时间 (15:00)

我应该：
1. 指出时间错误
2. 请用户确认正确的时间

### Assistant
我发现时间设置有个问题：

❌ **开始时间**: 明天 15:00 (下午 3 点)
❌ **结束时间**: 明天 14:00 (下午 2 点)

结束时间不能早于开始时间哦！请问：
- 是 **15:00 到 18:00** (3 小时)？
- 还是 **14:00 到 17:00** (3 小时)？

请确认正确的时间范围~ 😊

```json
    "code": -32000,
```

```json
{

  }

```

## Agent 行为准则

### ✅ 应该做的
1. **主动收集信息**: 引导用户提供必需参数
2. **验证输入**: 检查时间范围、内容长度等
3. **友好提示**: 使用 emoji 和清晰格式
4. **错误处理**: 明确告知错误原因和解决方案
5. **后续引导**: 创建成功后提供下一步建议

### ❌ 不应该做的
1. ❌ 不验证就调用 API
2. ❌ 直接抛出原始错误信息
3. ❌ 不解释时间戳转换
4. ❌ 创建成功后不提供后续选项
